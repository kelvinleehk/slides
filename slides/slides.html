What is Testable Code?

In short:
Testable is you code are organized in a way that can be easily tested through unit testing.

What Testable Code's NOT?

Code that you need to test. (Yeah you've heard right)

To be precise

Testable code is more a design pattern than a standard / requirement that keeps your code clean, decoupled, recyclable and structured in a concise manner.


It's the core of Test Driven Development (TDD)

How to make code testable?

How to make code testable?

  • Separation of concerns
  • Object-oriented code design
  • Loose coupling / dependency injection
  • Elimination of globals
  • API naming conventions
  • Clear code and documentation

Separation of concerns

Grand example:
Model-View-Controller (MVC)

Separation of concerns

Imagine when you have some of your Balance Sheet function logic and calcuation somewhere inside of your view that shows the report, and that you had to make some changes to the net profit calcuation in your controller logic.

the result will always be wrong!

Separation of concerns

Not only separating the concerns will help keep code highly maintainable (you know exactly where to look instead of doing a full codebase search), it will greatly help reduce code regression, thus more efficient coding workflow.

Separation of concerns

Don't try to have your functions to do too many things at the same time.

Loose coupling

/ Dependency Injection

while performing unit testing, functions will be run and tested individually instead of having the whole boostrap process of your site or application to run.

Which means by the time the function is being tested, not all the expected dependency will be made available.
